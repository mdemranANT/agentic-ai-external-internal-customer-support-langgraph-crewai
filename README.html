What it does

A user logs in via the Streamlit UI (or CLI / API) and asks a question in plain English. The system:

  1. Authenticates — identifies user as Customer or Employee (with loyalty tier / department).
  2. Classifies the query — routes to one of 8 departments + detects sentiment (positive / neutral / negative) + severity (low / medium / high). Uses conversation history for follow-up understanding.
  3. Routes smartly — only HIGH-severity negative queries → CrewAI escalation crew (3 specialist agents). Mild/medium complaints go through RAG so the department agent can help. Unrecognisable → escalate.
  4. Retrieves relevant FAQ snippets from a ChromaDB knowledge base (200 vectors, 8 departments).
  5. Augments with tool calls — order lookup, product search, loyalty points, department listing.
  6. Generates a grounded answer using GPT-4o with the retrieved context + tool results + user profile.
  7. Reflects on its own answer (quality score 1-10). If score < 7, regenerates up to 3 retries.
  8. Transfers across departments — detects when a query belongs to another department and suggests a cross-department handoff via [SUGGEST_TRANSFER] tags, with a one-click transfer in the Streamlit UI.
  9. Remembers conversation per user via SQLite-backed persistent memory.
  10. Follows up — asks "Is there anything else I can help you with?" and handles farewell responses gracefully.

Stretch Goals Implemented

# Feature Details
1 Multi-user persistent memory SQLite checkpointer (shop4you_memory.db). Each user gets an isolated thread (user_{id}).
2 Escalation with CrewAI investigation Negative sentiment or unknown department → 3-agent CrewAI crew (Complaint Analyst → Account Investigator → Resolution Specialist) produces a structured escalation report with reference number.
3 Extra departments + custom workflows 8 departments (4 customer, 4 employee). 6 tools for product search, loyalty lookup, order lookup, KB search, department listing, and escalation.
4 REST API + Streamlit UI deployment FastAPI service with 6 endpoints (including cross-department transfer) + Streamlit chat UI with login, user context, transfer buttons, and farewell handling.

The 7 AI Agents

LangGraph Orchestration Agents (agents.py)

# Agent Node Role
1 Classifier classifier_agent GPT-4o structured output — detects department, sentiment, severity & reasoning
2 Department RAG department_rag_agent Retrieves context from ChromaDB, calls tools (product / loyalty / order), generates answers
3 Reflection reflection_agent Quality-checks answers (score 1–10). Regenerates with feedback if < 7. Up to 2 retries
4 Escalation escalation_agent Generates ESC-XXXXX reference, triggers CrewAI crew for high-severity cases

CrewAI Escalation Crew (escalation_crew.py) — sequential pipeline

# Agent Framework Role
5 Complaint Analyst CrewAI Extracts core issue, urgency, emotional state, key details (order IDs, dates, amounts)
6 Account Investigator CrewAI Builds customer context, identifies responsible dept, flags VIP / repeat / red-flag issues
7 Resolution Specialist CrewAI Proposes resolution + goodwill gesture, sets response time, writes final escalation report

Agents 1–4 (LangGraph) handle ~95% of queries. When Agent 1 detects negative sentiment with HIGH severity (angry, abusive, threatening), Agent 4 hands off to Agents 5 → 6 → 7 which run as a sequential CrewAI crew. Mild/medium negative queries stay in RAG so the department agent can resolve them directly.

Quick Start

1. Clone & create environment

conda env create -f environment.yml
conda activate shop4you

Or with pip:

python -m venv .venv && .venv\Scripts\activate   # Windows
pip install -r requirements.txt

2. Set up API keys

Copy .env.example to .env and fill in your keys:

OPENAI_API_KEY=your_openai_api_key
TAVILY_API_KEY=your_tavily_api_key   # optional, not needed for core flow

3. Generate FAQ data (already done — skip if data/ has files)

python generate_data.py

4. Ingest into ChromaDB (already done — skip if knowledge_base/chroma/ exists)

python vector_store.py

5. Run the Streamlit UI recommended for demo

python -m streamlit run streamlit_app.py --server.port 8502
# Open: http://localhost:8502

Login with any demo email (e.g. sarah@gmail.com, priya@shop4you.com) and start chatting.

6. Run the CLI

python main.py

7. Run the API

python -m uvicorn app:app --reload
# Swagger UI: http://localhost:8000/docs

8. Run tests

pytest test_rag.py test_agent.py test_memory_isolation.py -v # 13 pytest tests
python test_orchestration.py                            # 70 orchestration tests (needs API key, ~5 min)
python demo_customer_showcase.py                        # 10-scenario Rich demo

Smart Routing & Orchestration

Severity-Based Escalation

Instead of escalating all negative-sentiment queries, the system uses a 3-tier severity model:

Severity Behaviour Examples
Low RAG handles normally “Could be better”, “Not impressed”
Medium RAG with empathy-first tone “Disappointed with quality”, “Upset about delay”
High → CrewAI escalation crew “I will sue!”, “Worst company ever!”, threats

Cross-Department Handoff

When a department agent detects that a query belongs to another department:

  1. The agent embeds a [SUGGEST_TRANSFER: dept_key] tag in its response
  2. The orchestrator parses the tag and populates suggested_transfer
  3. In Streamlit: a transfer button appears for the user to accept
  4. In FastAPI: the /transfer endpoint re-runs the query in the target department, skipping classification

Fallback: if no tag is present but the response mentions another department in a transfer context (e.g. “reach out to Shipping”), the orchestrator uses best-match scoring to detect the intended department.

Transfer/Connect Handling

22 transfer keywords are detected (e.g. “connect me to”, “pass me to”, “put me through”). When a user asks to be connected to a department, the agent introduces itself and asks how it can help — it never redirects.

Conversation-Aware Classification

The classifier receives the last 3 turns of conversation history, so follow-up requests like “pass me to customer service” (after a shipping conversation) correctly route to shipping, not to an unknown department.

Generic Term Handling

“Customer service”, “support”, “agent”, “representative” are mapped to the contextually appropriate department (or orders_returns as default when no history exists). These never generate an unknown classification.

Configuration

All tuneable values live in config.py:

Setting Default Notes
LLM_MODEL gpt-4o primary model
LLM_MODEL_FALLBACK gpt-4.1 fallback option
LLM_TEMPERATURE 0 deterministic outputs
EMBEDDING_MODEL text-embedding-3-small embedding model
MAX_REFLECTION_ATTEMPTS 3 max quality-check retries
CREWAI_ESCALATION_ENABLED true enable CrewAI crew on escalation (env var: CREWAI_ESCALATION)
RAG_TOP_K 3 docs per retrieval
RAG_SCORE_THRESHOLD 0.2 minimum similarity
LOG_LEVEL INFO set via env var LOG_LEVEL

Logging

The agent logs classification decisions, routing, reflection scores, and escalations via Python's logging module (logger name: shop4you). Set LOG_LEVEL=DEBUG in .env for verbose output.
