Shop4You Capstone — Complete Project Steps

Stretch 1: Multi-User Memory Stretch 2: Escalation Form Stretch 3: More Departments Stretch 4: FastAPI Deploy

Phase 1: Setup & Data

Week 1

1Create Project Directory Structure

Create folders: data/, knowledge_base/, agents/, tools/, config/, api/. Initialize virtual environment and .gitignore.

2Finalize Departments (8+ total) Stretch 3

  • External: Orders & Returns, Billing/Payment, Shipping/Delivery, Product Inquiries
  • Internal: HR, IT Helpdesk, Operations, Loyalty Program / Employee Benefits

Define each department's scope, audience, and tone.

3Generate Synthetic QA Datasets

Use an LLM to generate 10–15 QA pairs per department (80–120 total). Each entry: question, answer, tags, last_updated, confidence_score.

4Store Datasets as JSON

One JSON file per department with schema: department, doc_id, entries[].

5Set Up Environment Variables

Create .env with OPENAI_API_KEY, TAVILY_API_KEY. Use python-dotenv.

6Install All Dependencies

langchain, langchain-openai, langchain-community, langgraph
langchain-chroma, langgraph-checkpoint-sqlite
fastapi, uvicorn, pydantic, python-dotenv, rich, tiktoken

7Create Embeddings & Ingest into ChromaDB

Embed all QA docs using OpenAIEmbeddings(model='text-embedding-3-small'). Store in ChromaDB with department metadata. Persist to ./knowledge_base.

Phase 2: Core RAG Pipeline

Week 2

8Build Retriever with Metadata Filtering

kbase_db.as_retriever() with similarity_score_threshold. Dynamic filter={"department": dept} per query.

9Create RAG Prompt Templates

Per-department prompt templates using ChatPromptTemplate. Include customer query + retrieved context. Fallback message for unanswerable queries.

10Build RAG Worker Node

Input: department + user query → retrieve context → inject into prompt → LLM generates answer. Output: formatted response with citations.

11Test RAG Independently

Verify each department retrieves correct documents. Test edge cases: ambiguous queries, empty retrieval, cross-department queries.

Phase 3: Router & Agentic Logic

Week 3

12Build Router Agent

LLM classifies department + sentiment using with_structured_output(). Pydantic models for QueryCategory and QuerySentiment.

13Build Routing Logic

add_conditional_edges(): Negative sentiment OR Unknown department → Escalation. Valid department + Positive/Neutral → Department RAG node.

14Build Escalation Node with Form Stretch 2

Collect user details: name, phone, email. Display confirmation. Simulate sending email/WhatsApp notification. Store in escalation_info state.

15Build Reflection / Self-Check Loop

After generating RAG answer: check retrieval relevance, hallucination risk, policy compliance. Retry up to MAX_ATTEMPTS = 3.

16Build Custom Workflows for Extra Departments Stretch 3

Product Inquiries: product search tool, stock check. Loyalty Program: points lookup, rewards info. Specialized @tool functions per department.

17Add Multi-User Conversational Memory Stretch 1

SqliteSaver.from_conn_string("shop4you_memory.db"). Unique thread_id per user. Conversations persist across sessions. User isolation guaranteed.

18Define State Schema

class Shop4YouState(TypedDict):
    messages: Annotated[list, add_messages]
    customer_query: str
    department: str
    sentiment: str
    retrieved_context: str
    plan: List[str]
    past_steps: Annotated[List[Tuple], operator.add]
    reflection_count: int
    final_response: str
    user_id: str
    escalation_info: dict

19Wire the Full LangGraph

START
  → Step 0: classify_query (department, sentiment, severity)
  → route_query (conditional — one path only)
      ├─ IF valid dept (or low/med negative):
      │    → Step 1a: department_rag (ChromaDB + tools + GPT-4o)
      │    → Step 2:  reflect (quality score ≥7 → done, else retry up to 3×)
      │    → END
      └─ IF negative+HIGH severity OR unknown dept:
           → Step 1b: escalate (CrewAI 3-agent crew → escalation_agent)
           → END

Phase 4: API & Deployment Stretch 4

Week 4 (first half)

20Create FastAPI App

POST /chat endpoint: accepts { "query": "...", "user_id": "..." }. Returns { "response": "...", "department": "...", "sentiment": "..." }. Swagger UI at /docs.

21Integrate Agent with API

API calls compiled LangGraph agent. Passes user_id as thread_id for memory. Returns structured response.

22Add Session Management

GET /history/{user_id} — retrieve conversation history. DELETE /history/{user_id} — clear a user's session.

23Deploy the Web Service

Run with uvicorn api.main:app --reload. Test via Swagger UI, curl, or Postman. Optional: deploy to cloud (Render, Railway, AWS).

Phase 5: Testing & Polish

Week 4 (second half)

24End-to-End Testing

Test 10+ sample queries covering:

  • ✔ Each department (8+ queries, one per dept)
  • ✔ Negative sentiment → escalation with form collection
  • ✔ Unknown department → escalation
  • ✔ Multi-turn conversations (same user returns, context recalled)
  • ✔ Different users don't see each other's history
  • ✔ API endpoint testing via Swagger/Postman
  • ✔ Reflection loop triggers on low-quality retrieval
  • ✔ Severity-based escalation (only HIGH severity escalates, not all negative)
  • ✔ Cross-department handoff (SUGGEST_TRANSFER tag + fallback detection)
  • ✔ Transfer keyword detection (22 phrases including "pass me to", "connect me")
  • ✔ Generic term handling ("customer service" → contextual department)
  • ✔ Conversation-aware classification (last 3 turns for follow-up context)
  • ✔ No "contact customer service" in responses (KB data fixed + RAG override)
  • ✔ 70 orchestration tests passed across 12 categories

25Add Logging and Metrics

Query logs with timestamps. Retrieval precision. Escalation rate. Response latency tracking.

26Verify Memory Isolation

User A and User B have independent conversations. "What did we discuss?" returns only that user's history. SQLite persistence survives app restarts.
